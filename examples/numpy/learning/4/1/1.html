배열 형태 일치

   넘파이에서는 산술 연산자를 사용해 배열 간 연산을 수행할 수 있습니다. 같은 위치의 요소끼리 연산하기 때문에 요소별 연산(element-wise operation)이라고도 부릅니다.


   배열 간 연산을 위해서는 연산하고자 하는 배열들의 형태가 같아야 합니다. 차원의 크기와 축별 크기가 동일하지 않으면 ‘ValueError’가 발생합니다.






Code A

   array1과 array2에 저장된 (2, 3) 형태의 2차원 배열을 서로 더해 봅시다.

Code A 를 실행하여 출력 결과를 확인하세요.




Code B

   array1과 array2에 저장된 (2, 3) 형태의 2차원 배열을 서로 곱해 봅시다.

Code B 를 실행하여 출력 결과를 확인하세요.


2

데이터 유형 일치

   또한 연산하고자 하는 배열들의 데이터 유형이 같아야 합니다. 문자형과 숫자형처럼 서로 다른 데이터 유형으로 이루어진 배열을 연산하려 하면 ‘TypeError’가 발생합니다.


   만약 int8과 float16처럼 서로 다른 숫자형 배열을 연산할 때는, 연산 전에 두 데이터 유형을 모두 표현할 수 있는 공통 데이터 유형으로 변환됩니다. 이를 ‘유형 통일(type promotion)’이라고 부릅니다.






Code C

   array3과 array4에 저장된 배열은 같은 형태이지만, 데이터 유형이 다릅니다. 두 배열을 서로 더해 봅시다.

Code C 를 실행하여 출력 결과를 확인하세요.




Code D

   array5에 저장된 배열은 정수형, array6에 저장된 배열은 실수형입니다. 동일한 형태의 3차원 배열을 서로 곱해 보고, 반환된 배열의 데이터 유형을 확인해 봅시다.

Code D 를 실행하여 출력 결과를 확인하세요.

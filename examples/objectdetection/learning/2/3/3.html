EPOCH

에포크는 훈련 데이터 전체를 한 번 완전히 통과한 횟수를 의미합니다.

에포크 동안 모델은 모든 데이터를 한 번씩 학습합니다.

예를 들어, 데이터셋이 1,000개 샘플로 이루어져 있고 배치 크기(batch size)가 100이라면, 10번의 이테레이션(iteration)이 모여 1 에포크를 구성합니다.

일반적으로 모델이 효과적으로 학습하려면 여러 번의 에포크를 거쳐야 합니다.

Box Loss

Box Loss는 모델이 예측한 바운딩 박스와 실제 정답 바운딩 박스 간의 차이를 평가합니다.

바운딩 박스의 위치와 크기를 얼마나 정확히 예측했는지를 판단합니다.

IoU(Intersection over Union)나 L1/L2 손실 값을 사용해 계산합니다.

훈련이 진행될수록 Box Loss는 점차 감소하며, 모델이 바운딩 박스를 더 정확히 예측하게 됩니다.

Cls Loss

Cls Loss는 바운딩 박스 안의 객체가 어떤 클래스인지 예측하는 정확도를 평가합니다.

객체 검출에서 모델이 클래스 분류를 올바르게 수행하도록 학습시킵니다.

주로 Cross Entropy Loss를 사용해 손실 값을 계산합니다.

훈련이 진행될수록 Cls Loss는 감소하며, 이는 모델이 클래스를 더 정확히 분류하고 있음을 나타냅니다.

DFL Loss

DFL Loss(Distribution Focal Loss)는 객체 검출에서 바운딩 박스의 좌표를 더 정확히 예측하기 위해 사용되는 손실 함수입니다.

DFL은 각 좌표값에 대한 분포(Distribution)를 학습시켜 좌표 예측을 더욱 정밀하게 만듭니다.

기존의 바운딩 박스 회귀 방식이 좌표 값을 직접 예측하는 것과 달리, DFL은 좌표가 연속적인 분포로 표현된다고 가정합니다.

이 방식을 통해 예측값에 대한 불확실성을 모델링할 수 있으며, 더 높은 정확도를 제공합니다.

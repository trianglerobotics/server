객체 지향 프로그래밍이란?

   객체 지향 프로그래밍(OOP, Object Oriented Programming)은 프로그램을 객체라는 독립적인 단위로 나누어서 설계하는 프로그래밍 방식입니다.

   다양한 데이터와 수행해야 할 작업들을 하나로 묶어 집합체로 만들고, 이러한 집합체를 여러 개 만들어 각각의 집합체들이 서로 협력하여 하나의 프로그램으로 작동하게 합니다.

   이러한 프로그래밍 방식은 복잡하고 방대한 분량의 프로그램도 단순한 구조로 만들 수 있습니다. 이 덕분에 프로그램을 유지 보수하는 데 매우 효과적입니다.




2

클래스(class)란?

   클래스는 데이터와 함수를 묶어서 관리하는 집합체입니다. 객체 지향 프로그래밍의 핵심 요소로, 클래스를 기반으로 다양한 객체들을 생성하여 프로그램을 작성할 수 있습니다. 특정 클래스에서 생성된 객체들은 인스턴스(instance)라고 부릅니다.

   클래스와 객체는 집을 짓는 과정에 비유할 수 있습니다. 클래스가 집을 짓기 위한 설계도라면, 설계도에 맞추어 제작된 기둥과 창문과 같은 실제 구성 요소가 객체에 해당합니다.




클래스 기본 구조
  • class: 클래스 정의를 시작하는 예약어입니다.
  • 클래스명: 클래스 이름을 지정합니다. 일반적으로 대문자로 시작합니다.
  • 변수 정의: 클래스 내부에서 사용되는 변수를 저장할 수 있습니다. 멤버 변수(member variable)라고 부릅니다.
  • 함수 정의: 클래스 내부에서 작동하는 함수를 정의할 수 있습니다. 메소드(method)라고 부릅니다.

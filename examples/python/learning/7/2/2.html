클래스 변수(class variable)

   클래스 변수는 멤버 변수 중 하나로 클래스 내부에서 정의되는 변수입니다. 클래스에 속해 있으며, 일반적인 변수와 동일하게 기능합니다. 따로 수정하지 않으면 데이터가 변하지 않아 클래스 작동에 필요한 데이터를 저장해 놓을 때 사용합니다.

   예를 들어 연필을 만드는 공장이 있다고 가정해 봅시다. 어떤 연필을 생산하든 연필의 길이나 지름 같은 기본적인 설계 규격은 바뀌지 않습니다. 따라서 연필의 규격을 한 번만 정해 놓으면, 어떤 연필이든 정해 놓은 규격을 참조하여 생산됩니다.

   이처럼 클래스 변수는 클래스에서 공통적으로 사용되는 데이터를 저장하고, 클래스로부터 생성되는 객체가 그 값을 참조하도록 합니다.




클래스 변수 사용 예시
  • class: 클래스 정의를 시작하는 예약어입니다.
  • Pencil: 클래스명입니다.
  • length···: 클래스 내부에 저장된 클래스 변수입니다.
  • Pencil.length···: 호출된 클래스 변수입니다. ‘클래스명.변수명’ 형식으로 사용할 수 있습니다.

인스턴스 변수(instance variable)

   인스턴스 변수는 멤버 변수 중 하나로 클래스로부터 생성된 객체가 독립적인 값을 가지도록 정의된 변수입니다. 클래스 변수가 모든 객체가 공유하는 데이터라면, 인스턴스 변수는 특정 객체에만 속하는 데이터입니다.

   연필 공장에서 연필을 만들 때 정해진 규격에 따라 만들더라도, 몸통의 색깔이나 연필심의 종류는 달리할 수 있습니다. 만약 새로 제작할 연필에 분홍색 몸통과 HB 심을 사용해야 한다면, 그에 따라 생산 라인의 일부를 조정해야 합니다.

   이와 같이 인스턴스 변수를 통해 객체마다 고유한 값을 부여하고 독립적으로 관리할 수 있습니다. 연필 공장에서 생산되는 연필의 특징이 제품마다 다르듯이, 인스턴스 변수는 생성되는 객체마다 서로 다른 데이터를 저장할 수 있습니다.




인스턴스 변수 생성 메소드(생성자)
  • def: 함수 정의를 시작하는 예약어입니다. 생성자를 사용할 때도 이용합니다.
  • __init__: 객체의 초기 상태를 설정하는 메소드입니다. 보통 생성자로 불립니다.
  • self: 생성자의 필수 매개 변수입니다. 객체마다 데이터를 독립적으로 관리한다는 것을 의미합니다.
  • inst_n: 인스턴스 변수명을 매개 변수로 입력합니다. 객체를 생성하면 매개 변수로 입력한 인스턴스 변수에 데이터를 삽입할 수 있습니다.
  • self.para_n = para_n: 인스턴스 변수를 초기화하는 작업니다. 객체를 생성할 때마다 인스턴스 변수에 새로운 데이터를 입력 받기 위한 준비 과정입니다.


인스턴스 변수 사용 예시
  • new1.color···: 호출된 인스턴스 변수입니다. ‘객체명.인스턴스 변수명’ 형식으로 사용할 수 있습니다.
